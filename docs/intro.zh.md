# HHeader：一个开源的 Manifest V3 请求/响应头编辑器

> 按域名自定义、修改、删除 HTTP 请求头与响应头，支持变量、脚本、重定向、隐私模式、多端同步与可视化分析。

---

## 一、它是什么

**HHeader** 是一个开源的 Chrome 浏览器扩展（Manifest V3），定位类似 ModHeader，但更现代、功能更完整。它让你在浏览器层面按域名精细控制 HTTP **请求头（Request Header）** 和 **响应头（Response Header）**，无需改代码、无需代理、无需刷新页面即可即时生效。

核心使用场景：

- 给某个域名自动带上 `Authorization` / `X-API-Key`，省去每次手动填。
- 前端联调时伪造 `User-Agent`、`X-Forwarded-For` 测试后端分支逻辑。
- 临时移除或改写 `Set-Cookie` / `ETag` 排查缓存、登录态问题。
- 修复本地开发时的 CORS / CSP 限制，让接口能调通。
- 把请求重定向到 mock 服务，做隔离测试。

## 二、特性一览

### 1. Profile 配置系统

- 把规则分组到多个 **Profile（配置集）**，可独立启用/停用、拖拽排序、克隆、配色标签。
- 工具栏图标颜色会跟随当前启用的 Profile，一眼区分当前生效的是哪套规则。
- 支持 **JSON 导入 / 导出**，方便备份和分享配置。

### 2. 请求头 & 响应头编辑

绝大多数工具只关注「发出去」的请求头，而 HHeader 同时支持「回来」的响应头：

- **Add（添加）**：追加请求头（受 Chrome 保护的头会自动降级为 set，避免规则被拒）。
- **Modify（修改）**：覆盖已有请求头 / 响应头的值。
- **Filter（过滤）**：删除指定的请求头 / 响应头。
- 每条规则用「方向」开关区分 `请求 →` 还是 `响应 ←`。

### 3. 灵活的域名匹配

- **通配符**：`example.com` 与 `*.example.com` 都能匹配（自动转为 `||example.com`）。
- **正则**：直接写正则过滤整条 URL（Chrome 不支持的正则会自动跳过并给出提示）。
- **路径锚定**：支持 `/path` 形式的细粒度匹配。

### 4. 变量与动态占位符

- 自定义变量集中管理（如 `Token`、`Env`），规则里用 `{{变量名}}` 引用。
- 内置动态占位符：`{{TIMESTAMP}}`、`{{TIMESTAMP_MS}}`、`{{ISODATE}}`、`{{DATE}}`、`{{UUID}}`、`{{RANDOM}}`。
- 带动态占位符时，扩展会每分钟自动重新同步一次规则，保证值不过期。

### 5. 脚本化规则（沙箱隔离）

- 可以写一个 JS 函数 `(request, ctx) => ({ headers: [{ name, value }] })`，根据请求上下文动态计算要注入的头部。
- **MV3 限制下的安全实现**：Chrome 禁止扩展页面用 `eval`/`new Function`，且 DNR 规则是声明式的、无法在请求时执行 JS。HHeader 把脚本放到一个 **沙箱 iframe** 中执行（CSP 允许脚本但禁止同源），并把计算结果快照成静态规则。脚本里可用 `md5`、`btoa` 等辅助函数。

### 6. 请求重定向（Redirect）

- 把匹配到的请求重定向到另一个 URL（仅限 `http(s)://`）。
- 适合把线上接口临时指到本地 mock，或把某个静态资源换成调试版。

### 7. 隐私模式（Stealth）

- 一键开启：自动剥离 `Referer`、`User-Agent` 指纹、`ETag`、`Set-Cookie` 等可被追踪的头部。
- 隐私规则优先级低于用户规则（隐私 1–4，用户基础 100），所以你自己的规则永远赢。

### 8. 预设库（Presets）

- 内置常用组合：模拟 Googlebot / Bingbot 爬虫、iPhone / Android 移动端、无缓存调试、安全加固等，一键套用到 Profile。

### 9. 多端同步

配置可以在多台设备间同步，支持多种通道：

- **本地文件**：导出/导入 JSON。
- **GitHub Gist**：推送 / 拉取。
- **WebDAV**：连接任意 WebDAV 服务器（如 Nextcloud、群晖）。
- **私有 GitHub 仓库**：通过 Contents API 读写指定路径的文件。
- Sync 面板用左侧竖排 Tab 切换各通道，一次只看一个，干净直观。

### 10. 调试日志 + 一键复制

- 开发模式下自动记录每条命中的规则（URL、方法、改了哪个头、改成了什么）。
- 支持把某次请求 **一键复制为 cURL / Fetch / Python** 代码，方便在终端或脚本里复现。

### 11. 数据分析面板（Analytics）

- 内置基于 ECharts 的可视化面板（按需懒加载，不拖慢弹窗启动）：
  - 总命中数、CORS 修复数、请求 / 响应占比。
  - 高频被修改的头部 Top 榜。
  - 24 小时命中趋势曲线。
- 统计写在 `chrome.storage.local`，跨扩展重启累计。

### 12. 资源类型 / 请求方法过滤

- 每条规则可限定只作用于特定资源类型（document / xhr / script / image …）和 HTTP 方法（GET / POST …），避免误伤。

### 13. 精致的 UI / UX

- **深色模式**：自动跟随系统，也可手动切换 浅 / 深 / 自动。
- **拖拽排序**：Profile 之间、规则之间都能拖。
- **右键菜单**：在任意页面「将当前域名加入 HHeader」一键建规则。
- **图标颜色同步**：工具栏图标跟着当前 Profile 配色走。
- **徽章计数**：工具栏角标显示当前生效的规则数量。

## 三、技术架构亮点

- **Manifest V3 + `declarativeNetRequest`**：规则以声明式动态规则注册，性能高、权限可控、符合 Chrome 未来方向。
- **规则引擎映射清晰**：Add→`append`、Modify→`set`、Filter→`remove`；响应头走 `responseHeaders`、请求头走 `requestHeaders`；重定向走 `redirect` action。
- **优先级设计**：用户规则基础优先级 100，隐私规则 1–4，确保用户规则始终覆盖隐私规则。
- **脚本沙箱**：用 manifest `sandbox` 页面隔离执行用户 JS，绕开 MV3 的 CSP 限制，又不污染扩展主页面。
- **性能优化**：ECharts 按需只引所需图表/组件，并通过 `defineAsyncComponent` 代码分割，主弹窗体积维持在 ~102KB（gzip 39KB），统计页单独 ~524KB 仅在打开时加载。
- **测试与验证**：62 个单元测试（规则引擎 / 变量 / 预设 / 过滤）全部通过；另有基于 Playwright 的 **e2e 验证脚本**（`e2e/run.mjs`），真实加载扩展并对本地服务断言 6 类规则是否生效。
- **中英双语文档**：`README.md`（中文）与 `README.en.md`（英文）完整对照。

## 五、快速开始

```bash
git clone <repo-url>
cd hheader
npm install
npm run build
```

然后在 Chrome 打开 `chrome://extensions` → 开启「开发者模式」→ 「加载已解压的扩展程序」→ 选择 `dist/` 目录即可。

编写规则：点击工具栏图标，新建 Profile，添加规则，填域名、选方向（请求/响应）、选操作（添加/修改/过滤），保存后即时生效。

## 六、结语

HHeader 把「浏览器层 HTTP 头控制」这件小事做扎实了：不止改请求头，也能改响应头；不止静态值，也能用变量和脚本动态算；不止单机用，也能多端同步；还顺手把调试、隐私、可视化都补上了。最重要的是——**它开源、透明、跑在最新的 Manifest V3 上**。

如果你也经常被跨域、鉴权头、缓存、移动端模拟这些问题困扰，不妨试试 HHeader。
